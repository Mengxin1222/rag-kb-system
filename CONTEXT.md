# RAG 知识库问答系统

基于 RAG 技术的通用知识库问答系统。管理员创建知识库并上传文档，普通用户选择知识库提问，系统通过向量检索和大语言模型生成回答。

## Language

**Knowledge Base (知识库)**:
An isolated collection of documents. Each knowledge base has its own vector index and retrieval scope.
_Avoid_: 资料库, project, workspace

**Document (文档)**:
A file uploaded by an Admin into a Knowledge Base. Can be PDF, Word, PPT, Excel, TXT, or Markdown. Each Document has a Processing Status (待处理 → 处理中 → 已就绪 → 失败). After auto-chunking, the Admin can review and manually adjust Chunk boundaries via the Chunk Editor before finalizing. Individual Documents can override the KB's default Chunking Strategy. Deleting a Document cascades to remove all its Chunks from the vector database. Re-uploading replaces the old Document and its Chunks.
_Avoid_: File, attachment, resource

**Chunk (切片)**:
A section of a Document produced by applying a configurable Chunking Strategy. The atomic unit of retrieval — chunks are embedded and stored in the vector database. Each Chunk records its source page number(s) for citation. Chunks are tagged with content types (code, link, image) detected during splitting.
_Avoid_: Segment, fragment, block

**Chunking Strategy (切片策略)**:
A configurable set of rules that determine how a Document is split into Chunks. Three methods: heading-based (Markdown `#` hierarchy), character-based (recursive separator splitting), and semantic (embedding similarity valleys). Fenced code blocks are protected from being split mid-block; oversized code blocks are split at function/class boundaries via AST (Python) or blank lines (other languages). Other settings: max character limit, adjacent Chunk overlap, ordered separators for recursive splitting. Set at the Knowledge Base level as a default; individual Documents can override. Default: heading-based, split on level-2 headings (##), no overlap.
_Avoid_: Splitting config, chunk settings

**Content Tags (内容标签)**:
Per-chunk metadata indicating whether the chunk contains code (`code`), hyperlinks (`link`), or images (`image`). Detected via regex during chunking and stored as a JSON array. Used by the frontend for rendering decisions and the search page for filtering.
_Avoid_: content type, content flags

**Admin (管理员)**:
A user who can create knowledge bases, upload documents, manage users, and configure system parameters.
_Avoid_: Owner, manager

**User (普通用户)**:
A user who can browse knowledge bases, search across them, and ask questions. Read-only access.
_Avoid_: Viewer, member, guest

**Query (提问)**:
A question submitted by a User against a specific Knowledge Base.
_Avoid_: Question, prompt, input

**Answer (回答)**:
The response generated by the LLM based on retrieved Chunks. Includes source citations (document name + page number).
_Avoid_: Response, reply, output

**Conversation (对话)**:
A thread of Queries and Answers within a Knowledge Base, with multi-turn context preserved. The maximum number of rounds retained is configurable. When the history exceeds the limit, earlier rounds are compressed into a summary via Context Compression. Partial answers are saved if the SSE connection is interrupted.
_Avoid_: Session, thread, chat

**Context Compression (上下文压缩)**:
A feature that summarizes earlier conversation rounds into a condensed form when the conversation history exceeds the configured round limit, preserving key information while reducing token usage. Enabled per Knowledge Base.
_Avoid_: Summarization, history compaction

**Retrieval (检索)**:
The hybrid process of finding the Top-K most relevant Chunks using both vector similarity search (K1 configurable) and BM25 keyword search (K2 configurable), fused via Reciprocal Rank Fusion (RRF). RRF constant K is configurable per Knowledge Base. After fusion, results are re-ranked (see Rerank).
_Avoid_: Search, match

**Rerank (重排序)**:
Re-scoring the retrieved Chunks using a dedicated reranking model to produce a refined Top-N before sending to the LLM. N is configurable per Knowledge Base.
_Avoid_: Reorder, rescore

**Chunk Search (切片搜索)**:
A standalone search feature that allows Users to search across multiple Knowledge Bases by keyword (BM25), vector similarity, or hybrid (both). Returns ranked Chunks with source document filenames, page numbers, and content tags. Supports multi-KB selection.
_Avoid_: Full-text search, global search

**Pipeline (文档处理管道)**:
The processing chain that converts an uploaded Document into Chunks ready for embedding. Non-Markdown formats are first converted to Markdown via MinerU. Processing runs asynchronously: upload returns immediately, the document progresses through statuses (待处理 → 处理中 → 已就绪 → 失败) in the background.
_Avoid_: Workflow, transformer
