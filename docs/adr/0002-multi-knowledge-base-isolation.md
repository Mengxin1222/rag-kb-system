# 多知识库隔离检索

系统支持管理员创建多个知识库，每个知识库拥有独立的向量索引。用户提问时必须选择目标知识库，检索范围仅限该知识库内的文档。

## Considered Options

- **单一知识库** — 所有文档混合在一起，无需选择，使用简单。但随着文档量增长，无关文档干扰检索精度，且无法区分不同主题域。
- **标签/分类过滤** — 不物理隔离，通过元数据标签过滤检索结果。实现简单但过滤逻辑依赖于标签准确性，且标签体系难以维护。
- **多知识库物理隔离** — 每个知识库独立的 ChromaDB Collection，检索天然隔离，精度最高。代价是需要显式选择知识库。

## Consequences

- 每个知识库在 ChromaDB 中对应一个独立的 Collection。
- 对话历史与知识库绑定，跨知识库的消息不会出现在同一对话中。
- 管理员对文档的管理操作（上传、删除）范围限定在单个知识库内。
